Following is a textual walk-through of the Balance Verification – getRedemptionAmount operation.

1.  Service Consumer invokes getRedemptionAmount operation of Balance Verification service to determine the redemption value of the chosen Plan Offering.

2.  If value of stackedQueued is received in request (identified by /planOffering/stackedQueued is NOT NULL)

    1.  Balance Verification component invoke [queryPlanOffering](../../catalog/Manage Product Offering/Product Offering Inquiry.md#queryplanoffering) operation of [Product Offering Inquiry](../../catalog/Manage Product Offering/Product Offering Inquiry.md) service to retrieve Plan Offering details for the received Plan Offering.  
        **Note:** specify /planOffering/planID = <from request message;  /planOffering/id\>,  
		/characteristic[name= 'MERCHANT_STATE']/value = <from request message; /characteristic[name= 'MERCHANT_STATE']/value\>,  
		/characteristic[name= 'PRODUCT_STATE']/value = <from request message; /characteristic[name= 'MERCHANT_STATE']/value\>,  
		/filterCriterias/value = <DISCOUNTS\>  
        **Note:** Retrieve following details of the Plan Offering(s)

        -   PlanOffering.id

            *   determined from request */planOffering/id*

        -   PlanOffering.category

            *   determined from response of queryPlanOffering; /planOffering/planCategory

        -   price

            *   determined from response of queryPlanOffering; /planOffering/compositePrice[priceType= '2' and priceSubType = '3']/price

        -   priceNetOfDiscountTaxAndProcessingFee

            *   determined from response of queryPlanOffering; /planOffering/compositePrice[priceType= '2' and priceSubType = '3']/priceNetOfDisctTaxProcessingFee

        -   bundledNetflixExists

            *   determined from response of queryPlanOffering; if /planOffering/planSpecifications[planSpecServiceType = 'Z0072'] exists, specify Y; otherwise specify N

3.  If value of stackedQueued is received in request and Plan Offering is Queued (identified by /planOffering/stackedQueued is NOT NULL and /planOffering/stackedQueued = 1: Queued)  
    **Note:** determine redemption value of Plan Offering in queue

    1.  Balance Verification component specifies the determined redemption amount.  
        **Note:** specify /planOffering/category = <from response of queryPlanOffering; /planOffering/planCategory\>,  
        /planOffering/redemptionAmount = <from response of queryPlanOffering; /planOffering/compositePrice[priceType= '2' and priceSubType = '3']/price\>,  
        /planOffering/netRedemptionAmount = <from response of queryPlanOffering; /planOffering/compositePrice[priceType= '2' and priceSubType = '3']/priceNetOfDisctTaxProcessingFee\>

4.  Otherwise  
    **Note:** determine redemption value of Plan Offering – Connectivity in stack

    1.  Balance Verification component retrieves the transaction routing details (Endpoint URL) for the destination instance of OCS. Refer [Routing – Outbound – Service Component to route](../../appendices/Appendix G.md#routing-outbound-service-component-to-route) for details.

    2.  Balance Verification component translates the request from CMM (Reliance SID) to the proprietary message model of OCS and invokes chargeableItemCheckCharge operation of ChargeableItemCharging service.

    3.  The response received from OCS is translated from the proprietary message model of OCS to the CMM (Reliance SID).

    4.  If bundledNetflixExists = 'Y' (from response of /planOffering/planSpecifications[planSpecServiceType = 'Z0072'] exists)  
        **Note:** applicable only when value of stackedQueued is received in request and bundledNetflixExists is determined  
        **Note:** determine redemption value of Netflix Flexi Top-up adjustment amount, if applicable

        1.  Balance Verification component determines the Subscribed NetflixOfferId and Bundled NetflixOfferId for the Order Reference Number.

            -   Balance Verification component invokes [retrieveDigitalServiceByPlanOffering](../../../digital/inventory/Manage Customer Information/Digital Service Configuration Inquiry.md#retrievedigitalservicebyplanoffering) operation of [Digital Service Configuration Inquiry](../../../digital/inventory/Manage Customer Information/Digital Service Configuration Inquiry.md) service  
                **Note:** specify /orderReferenceNumber = <from request message; /planOffering/transRefNumber\>, /customer/customerId = <from request message; /customerId\>

            -   From the retrieved list, identify the NetflixOfferId, startDate and expiryDate of digitalService

                *   subscribedNetflixOfferId = /digitalServices[serviceType='Z0072' and status = '2']/planSpecification/characteristics[name = 'CATEGORY']/value

                *   bundledNetflixOfferId = /digitalServices[serviceType='Z0072' and status = '2']/planSpecification/characteristics[name = 'BASE_CATEGORY']/value

                *   subscribedNetflixStartDate = /digitalServices[serviceType='Z0072' and status = '2']/startDate

                *   subscribedNetflixExpiryDate = /digitalServices[serviceType='Z0072' and status = '2']/endDate

        2.  If subscribedNetflixOfferId is NOT identified

            -   Netflix Flexi Top-up Amount for adjustment (netflixAdjustmentAmount) = 0

        3.  Otherwise (subscribedNetflixOfferId is identified)

            -   compare the bundledNetflixOfferId with the subscribedNetflixOfferId (read "effective" priority for the subscribedNetflixStartDate)

            -   if priority of the subscribedNetflixOfferId \> bundledNetflixOfferId  
                **Note:** indicates Customer has upgraded the NetflixOfferId

                *   calculate days of adjustment

                    -   numOfDaysforAdjustment (in days) = subscribedNetflixExpiryDate (consider DateTime) – value as specified in request message chargeableItemCheckCharge; /chargeableItem/consumptionDate) (consider DateTime)  
                        Use roundoff with scale = 0 and RoundingMode.DOWN

                *   calculate difference in slabs (read "effective" amount(s) for the subscribedNetflixStartDate)

                    -   deltaAmount = Amount for subscribedNetflixOfferId – Amount for the bundledNetflixOfferId

                *   calculate the Netflix Flexi Top-up Amount for adjustment

                    -   netflixAdjustmentAmount = numOfDaysforAdjustment * (deltaAmount/30)  
                        Use roundoff with scale = 2 and RoundingMode.HALF_UP

            -   Otherwise

                *   Netflix Flexi Top-up Amount for adjustment (netflixAdjustmentAmount) = 0

        4.  If Netflix Flexi Top-up Amount for adjustment (netflixAdjustmentAmount) \> 0

            -   Balance Verification component specifies the redemption amount for the JPL  
                **Note:** specify /characteristics[name = 'NETREDEEMVALJPL']/value = netflixAdjustmentAmount

5.  Balance Verification component returns the response to the invoking component.
