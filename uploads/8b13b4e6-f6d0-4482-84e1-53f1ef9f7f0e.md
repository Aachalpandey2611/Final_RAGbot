Following is a textual walk-through of the EWallet Management – getEWalletBalance operation.

1.  Service Consumer invokes getEWalletBalance operation of EWallet Management service to get balance of the dealer wallet.
2.  On receiving the request, Media EWallet Management component retrieves the transaction routing details (Endpoint URL) for the destination instance of Estel. Refer [Routing – Outbound – Service Component to route](../../appendices/Appendix G.md#routing-outbound-service-component-to-route) for details.
3.  EWallet Management component translates the request from CMM (Reliance SID) to the proprietary message model of Estel and invoke the balance operation of the Merchant web service.
4.  On receiving response from Estel, EWallet Management component translates the message from the proprietary message model of Estel to the CMM (Reliance SID) and returns the wallet balance(s) to the invoking component.

| **Mediation Pattern** | Service Translator |

### Interaction Diagram

``` mermaid
sequenceDiagram
  autonumber
  participant RPOS as RPOS
  participant DAPI_EWM as DigitalApi (EWallet Management)
  participant ESTEL as Estel (RelianceServices)
  participant MSG_CN as Messaging (HANDLENOTIFICATION.IN.Queue)

  RPOS->>+DAPI_EWM: updateEWalletBalance
  Note over DAPI_EWM: Routing – Outbound – Service Provider with distributed deployment

  alt Transaction Type = Reserve
    DAPI_EWM->>+ESTEL: block
    ESTEL-->>-DAPI_EWM: block
  else Transaction Type = Release
    DAPI_EWM->>+ESTEL: unblock
    ESTEL-->>-DAPI_EWM: unblock
  else Transaction Type = Commit
    DAPI_EWM->>+ESTEL: blockCommit
    ESTEL-->>-DAPI_EWM: blockCommit
  end
  opt Characteristic[Name = "SEND_NOTIFICATION"]/Value != "N"
    DAPI_EWM->>MSG_CN: PUT
	Note over MSG_CN: Customer Notification – handleNotification
  end
  DAPI_EWM-->>-RPOS: updateEWalletBalance
```

Following is a textual walk-through of the EWallet Management – updateEWalletBalance operation.

1.  Service Consumer invokes updateEWalletBalance operation of EWallet Management service to either reserve or release the amount from the merchant's wallet.
2.  On receiving the request, EWallet Management component retrieves the transaction routing details (Endpoint URL) for the destination instance of Estel. Refer [Routing – Outbound – Service Component to route](../../appendices/Appendix G.md#routing-outbound-service-component-to-route) for details.
3.  EWallet Management component evaluates the transaction type and translates the request from CMM(Reliance SID) message to the proprietary message model of Estel.
4.  If the transaction type is "Reserve", EWallet Management component invokes the block operation of the Estel web service.  
    **Note:** The block operation reserves the amount based on the plan offering(s) in the merchant's wallet.
5.  If the transaction type is "Release", EWallet Management component invokes the unblock operation of the Estel web service.
6.  If the transaction type is "Commit", EWallet Management component invokes the blockCommit operation of the Estel web service.  
    **Note:** If EWallet Reservation Reference Id is specified in the request, the corresponding "Reserve" transaction which is initiated earlier is committed; otherwise the transaction commits the amount as specified.
7.  If Characteristic[Name = "SEND_NOTIFICATION"]/Value != "N"
    1.  EWallet Management component translates the message from the proprietary message model of Estel to the CMM (Reliance SID) and invokes [handleNotification](../../common/Customer Communication/Customer Notification.md#handlenotification) operation of [Customer Notification](../../common/Customer Communication/Customer Notification.md) service to submit a request for sending notification to the Merchant by putting a message to the messaging queue.
8.  EWallet Management component subsequently returns the response to the invoking component.


## transferEWalletBalance

### Interaction Diagram

``` mermaid
sequenceDiagram
  autonumber
  participant RRJD as RRJD SAP ERP (P447)
  participant DAPI_EWM as DigitalApi (EWallet Management)
  participant ESTEL as Estel (MerchantService)
  participant MSG_CN as Messaging (HANDLENOTIFICATION.IN.Queue)

  RRJD->>+DAPI_EWM: transferEWalletBalance
  Note over DAPI_EWM: Routing – Outbound – Service Provider with distributed deployment
  DAPI_EWM->>+ESTEL: wallettransfer
  ESTEL-->>-DAPI_EWM: wallettransfer
  loop 0, Count of Transfer Details
    DAPI_EWM->>MSG_CN: PUT
	Note over MSG_CN: Customer Notification – handleNotification
  end
  DAPI_EWM-->>-RRJD: transferEWalletBalance
```

Following is a textual walk-through of the EWallet Management – transferEWalletBalance operation.

1.  Service Consumer invokes transferEWalletBalance operation of EWallet Management service to transfer the balance from dealer's one wallet to another [downwards] (Wallet is created for each location of Dealer Organization and shared by all the Agents for the Location).
2.  On receiving the request, EWallet Management component retrieves the transaction routing details (Endpoint URL) for the destination instance of Estel. Refer [Routing – Outbound – Service Component to route](../../appendices/Appendix G.md#routing-outbound-service-component-to-route) for details.  
    **Note:** Specify Merchant Code (Role = D - Donor) to determine the Dealer Routing Zone, if not received.
3.  EWallet Management component translates the request from CMM(Reliance SID) to the proprietary message model of Estel and invokes the wallettransfer operation of the MerchantService web service.
4.  On receiving response from Estel, EWallet Management component translates the message from the proprietary message model of Estel to the CMM (Reliance SID).
5.  EWallet Management component iterates thru Count of Transfer Details and invokes [handleNotification](../../common/Customer Communication/Customer Notification.md#handlenotification) operation of [Customer Notification](../../common/Customer Communication/Customer Notification.md) service to submit a request for sending notification via. Mobile Number to the Dealer Agent(s), for successful transfer of the balance, by putting a message to the messaging queue.
6.  EWallet Management component returns the response to the invoking component.
