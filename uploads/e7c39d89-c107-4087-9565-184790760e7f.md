![Plan Subscription Management:configurePlanOffering](images/ServiceModel_configurePlanOffering.png)

Following is a textual walk-through of the Plan Subscription Management:configurePlanOffering service. This is implemented via. Orchestration Engine.

1.  MyJio invokes configurePlanOffering operation of Plan Subscription Management service for add-on pack add / remove or changing the Plan Offering for a Postpaid Customer.

2.  Plan Subscription Management Proxy component performs the basic semantic validations such as compliance of request message with structure including the mandatory / optional elements. If the validations fail, an error message / fault is returned to the invoking component.

3.  Plan Subscription Management Proxy component iterates through the line items (identified by /order/lineitems[]), creates an individual message for each request and puts the message to a messaging queue for processing. Only one line item is contained within each message.

4.  The acknowledgement message including the Change Order Reference Number is then returned to the invoking component.

5.  Plan Subscription Management component consumes the message from the messaging queue.
    ``` title="Note:"
    OE creates process instance(s) based on /order/orderRefNo and 
    /order/lineItems/planOfferings/planId (/order/orderRefNo +"_"+ 
    /order/lineItems/planOfferings/planId)
    ```

6.  Plan Subscription Management component invokes retrieveCustomerServiceConfiguration operation of Customer Configuration Inquiry service to retrieve the customer, list of accounts and products details and enrich the input message.
    ``` title="Note:"
    specify /serviceIdentifier/value =
    </order/lineItems[1]/planOfferings[1]/accounts[1]/products[1]/serviceId>,
    /serviceIdentifier/subCategory = <2>, 
	/includeAllServices = <true> and 
    /filterKey = <CFSADDON>
    ```
    ``` title="Note:"
    Routing Zone is identified as /routingZoneID
    ```

7.  If Routing Zone is not received in the request (HTTP Header, jioroute)

    1.  Enrich the value of retrieved Routing Zone in the request.

8.  If Payment Posting is NOT restricted and Payment Details is received in the request message and ((scenario is "Add-On Pack Add") or (scenario is "Bill Plan Change" and EWallet Reservation Reference Id is received in request message)) (identified by /lineItems/characteristics[name = 'POST_PAYMENT']/value = 'N' does NOT exists and /order/paymentDetails/totalAmountPayable exists and ((/order/lineItems/action = 01: ADD) or (/order/lineItems/action = 02: CHANGE and /order/lineItems/planOffering/ewalletReservationRefId exists)))
