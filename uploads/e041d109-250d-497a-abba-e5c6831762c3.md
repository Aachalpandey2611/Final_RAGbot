### Interaction Diagram

``` mermaid
sequenceDiagram
  autonumber
  participant CFP as CFP (Process Data Pooling)
  participant DAPI_PMM as DigitalApi (Pooled Member Maintenance)
  participant MZ as MZ(Recipient Server API)

  CFP->>+DAPI_PMM: maintainMember
  Note over DAPI_PMM: Routing – Outbound – Service Provider with distributed deployment
  
  alt transType = 01: Add Member
    DAPI_PMM->>+MZ: /insertRecipient
    MZ-->>-DAPI_PMM: /insertRecipient
  else transType = 02: Remove Member or 03: Remove Parent and Members
    DAPI_PMM->>+MZ: /deleteRecipient
    MZ-->>-DAPI_PMM: /deleteRecipient
  else transType = 04: Update IMSI of Parent or Member
    DAPI_PMM->>+MZ: /updateRecipient
    MZ-->>-DAPI_PMM: /updateRecipient
  end
  
  DAPI_PMM-->>-CFP: maintainMember
```

Following is a textual walk-through of the Pooled Member Maintenance:maintainMember operation.

1.  Upon receiving the request to maintainMember, Pooled Member Maintenance component retrieves the transaction routing details (Endpoint URL) for the destination instance of OCS. Refer [Routing – Outbound – Service Component to route](../../appendices/Appendix G.md#routing-outbound-service-component-to-route) for details.

2.  If transType = 01: Add Member
    1.  Pooled Member Maintenance component translates the request message from CMM (Reliance SID) to the proprietary message model of OCS and invokes /insertRecipient API, to register the pooled member.

3.  If transType = 02: Remove Member or 03: Remove Parent and Members
    1.  Pooled Member Maintenance component translates the request message from CMM (Reliance SID) to the proprietary message model of OCS and invokes /deleteRecipient API, to delete the pooled member or delete all the pooled members for the Parent.  
        **Note:** If records are NOT found in MZ, success is returned

4.  If transType = 04: Update IMSI of Parent or Member
    1.  Pooled Member Maintenance component translates the request message from CMM (Reliance SID) to the proprietary message model of OCS and invokes /updateRecipient API, to update the IMSI of pooled Parent or Member.

5.  On receiving response, Pooled Member Maintenance component translates the message from the proprietary message model of OCS to CMM (Reliance SID) and returns the response to the originating component.
