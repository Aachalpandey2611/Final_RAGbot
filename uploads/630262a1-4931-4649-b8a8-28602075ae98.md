### Interaction Diagram

``` mermaid
sequenceDiagram
  autonumber
  participant SC as Self Care

  SC->>+DAPI_RPI: findRefillPreference
  DAPI_RPI->>+EDIF: Get
  EDIF-->>-DAPI_RPI: Get
  DAPI_RPI-->>-SC: findRefillPreference
```

Following is a textual walk-through of the approach to implement Refill Preference Inquiry:findRefillPreference operation.

1.  On receiving the request, the Refill Preference Inquiry component translates the message from CMM (Reliance SID) to the proprietary message model of EDIF and creates the request to invoke the Get activity to retrieve the complete entries associated with provided key field(s) from an ActiveSpaces space.  
    **Note:** Upto two Refill Preferences exists for a Customer, one with Status = Active and other with P: Pending; When both are retrieved, Use the one with Status = Active for further processing based on the filter criteria.

2.  On receiving response from EDIF, Refill Preference Inquiry component translates the message from the proprietary message model of EDIF to the CMM (Reliance SID).

3.  Refill Preference Inquiry component then returns the list of Refill Preference(s) to the invoking component.
