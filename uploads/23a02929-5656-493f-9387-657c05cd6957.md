Following is a textual walk-through of the Integrated Recharge Reversal:performReversalFeasibility service.

1.  The consumer application invokes performReversalFeasibility operation of Integrated Recharge Reversal service for feasibility to reverse the balance replenishment transaction.

2.  Integrated Recharge Reversal component determines Count of Reversal transactions with status IN [1, 2] for the Customer in last 30 days (configurable) (cntLastMonth) and for the Customer in last 365 days (configurable) (cntlastYear)
    1.  If cntLastMonth \> 1 (configurable)
        -  Return /resultStatus/status = FAILED, /resultStatus/errorCode = 2011: Exceeds maximum number of allowed recharge reversals in last 30 days
    2.  Else If cntLastYear \> 3 (configurable)
        -  Return /resultStatus/status = FAILED, /resultStatus/errorCode = 2016: Exceeds maximum number of allowed recharge reversals in last 365 days
    3.  Otherwise
        -  Return /resultStatus/status = SUCCESS.

``` mermaid

  participant DAPI_BRR as DigitalApi (Balance Replenishment Reversal)

  IRP-->>-DAPI_BRR: maintainReversalStatus
```

Following is a textual walk-through of the Integrated Recharge Reversal:maintainReversalStatus service.

1.  Integrated Recharge Reversal component retrieves the Balance Replenishment Reversal transaction(s) based on request parameters.  
    ```title="Note:"
	if received /status = 1: Reversal Initiated,
        retrieve Distinct Balance Replenishment Reversal transaction(s) for
            latest transaction for the received Recharge Order Reference Number 
			(identified by /rechargeRefNo) and /customerId 
			or 
			received Balance Recovery Reference Number (identified by /orderRefNo)  
    Otherwise 
	    retrieve Balance Replenishment Reversal transaction(s) for 
		    received Balance Recovery Reference Number (identified by /orderRefNo)  
    ```
	**Note:** Balance Recovery Reference Number is unique.

2.  If multiple record(s) are retrieved,
    1.  Return /resultStatus/status = FAILED, /resultStatus/errorCode = 2014: Specify unique Balance Recovery Reference Number

3.  if received /status = 1: Reversal Initiated
    1.  If record is retrieved with status = 1: Reversal Initiated
        1.  if received /orderRefNo is same as retrieved /orderRefNo and received /rechargeRefNo is same as retrieved /rechargeRefNo.  
           **Note:** Indicates message is replayed again
            -  return /resultStatus/status = SUCCESS
        2.  if received /rechargeRefNo is same as retrieved /rechargeRefNo and received /orderRefNo is NOT same as retrieved /orderRefNo.
            -  return /resultStatus/status = FAILED, /resultStatus/errorCode = 2012: Recharge reversal in-progress for specified recharge reference number
        3.  if received /orderRefNo is same as retrieved /orderRefNo and received /rechargeRefNo is NOT same as retrieved /rechargeRefNo.
            -  return /resultStatus/status = FAILED, /resultStatus/errorCode = 2014: Specify unique Balance Recovery Reference Number
    2.  If record is retrieved with status = 2: Reversal Completed
        1.  return /resultStatus/status = FAILED, /resultStatus/errorCode = 2015: Recharge reversal completed for specified recharge reference number.
    3.  If record is retrieved with status = 3: Reversal Cancelled / Failed
        1.  if received /orderRefNo is same as retrieved /orderRefNo and received /rechargeRefNo is same as retrieved /rechargeRefNo.
            -  return /resultStatus/status = FAILED, /resultStatus/errorCode = 2014: Specify unique Balance Recovery Reference Number
        2.  if received /orderRefNo is same as retrieved /orderRefNo and received /rechargeRefNo is NOT same as retrieved /rechargeRefNo.
            -  return /resultStatus/status = FAILED, /resultStatus/errorCode = 2014: Specify unique Balance Recovery Reference Number
    4.  If no record is retrieved or (record is retrieved with status = 3: Reversal Cancelled / Failed and received /rechargeRefNo is same as retrieved /rechargeRefNo and received /orderRefNo is NOT same as retrieved /orderRefNo)
        1.  Integrated Recharge Reversal component determines Count of Reversal transactions with status IN [1, 2] for the Customer in last 30 days (configurable) (cntLastMonth) and for the Customer in last 365 days (configurable) (cntlastYear)
        2.  If cntLastMonth \> 1 (configurable)
            -  Return /resultStatus/status = FAILED, /resultStatus/errorCode = 2011: Exceeds maximum number of allowed recharge reversals in last 30 days
        3.  Else If cntLastYear \> 3 (configurable)
            -  Return /resultStatus/status = FAILED, /resultStatus/errorCode = 2016: Exceeds maximum number of allowed recharge reversals in last 365 days
        4.  Otherwise
            -  Create a record for Balance Replenishment Reversal with status = 1: Reversal Initiated for Customer and /resultStatus/status = SUCCESS.

4.  if received /status = 2: Reversal Completed
    1.  if record is retrieved with status = 1: Reversal Initiated
        1.  update record for Balance Replenishment Reversal with status = 2: Reversal Completed
        2.  return /resultStatus/status = SUCCESS.
    2.  If record is retrieved with status = 2: Reversal Completed  
        **Note:** Indicates message is replayed again
        1.  return /resultStatus/status = SUCCESS
    3.  If record is retrieved with status = 3: Reversal Cancelled / Failed
        1.  return /resultStatus/status = FAILED, /resultStatus/errorCode = 2014: Specify unique Balance Recovery Reference Number
    4.  Otherwise
        1.  return /resultStatus/status = FAILED, /resultStatus/errorCode = 2013: No transaction exists for the specified Balance Recovery Reference Number

5.  if received /status = 3: Reversal Cancelled / Failed
    1.  if retrieved status = 1: Reversal Initiated
        1.  update record for Balance Replenishment Revesal with status = 3: Reversal Cancelled / Failed
        2.  return /resultStatus/status = SUCCESS
    2.  If record is retrieved with status = 2: Reversal Completed
        1.  return /resultStatus/status = FAILED, /resultStatus/errorCode = 2015: Recharge reversal completed for specified recharge reference number.
    3.  If record is retrieved with status = 3: Reversal Cancelled / Failed  
        **Note:** Indicates message is replayed again
        1.  return /resultStatus/status = SUCCESS
    4.  Otherwise
        1.  return /resultStatus/status = FAILED, /resultStatus/errorCode = 2013: No transaction exists for the specified Balance Recovery Reference Number

6.  Otherwise
    1.  Return error 9998: Invalid request has been provided
