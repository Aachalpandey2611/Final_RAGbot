OCR Service Quick Start Guide

Setup & Installation
====================

1. Install OCR Dependencies:
   pip install -r requirements.txt

2. Database Setup:
   - Run migrations to create OCR tables
   - alembic upgrade head

3. Start Celery Worker (for async processing):
   celery -A app.core.celery_app worker -l info

4. Start API Server:
   uvicorn app.main:app --reload --port 8000

5. Monitor Celery Tasks:
   celery -A app.core.celery_app flower

Usage Examples
==============

A. Simple Text Extraction (Synchronous)
   
   POST /api/v1/ocr/extract HTTP/1.1
   Host: localhost:8000
   Authorization: Bearer YOUR_TOKEN
   
   Form Data:
   - file: <upload image.png>
   - provider: paddle
   - languages: en
   
   Response: Full OCR result with text, regions, confidence

B. Asynchronous Extraction

   POST /api/v1/ocr/extract-async HTTP/1.1
   
   Response: {"task_id": "job-uuid", "status": "pending"}
   
   Then poll:
   GET /api/v1/ocr/job/job-uuid HTTP/1.1
   
   Response: Updated status and results when ready

C. Batch Processing

   POST /api/v1/ocr/batch HTTP/1.1
   
   Form Data:
   - files: <multiple files>
   - task_name: "Batch Job Name"
   - provider: doctr
   
   Response: {"batch_id": "batch-uuid", "status": "pending"}
   
   Monitor:
   GET /api/v1/ocr/batch/batch-uuid HTTP/1.1

D. Check Available Providers

   GET /api/v1/ocr/providers
   
   Response: {"providers": ["paddle", "tesseract", "doctr"]}

E. List Supported Languages

   GET /api/v1/ocr/languages
   
   Response: List of language codes and names

Performance Tips
================

1. Cache Hits: Process same file multiple times = <100ms
2. GPU Mode: 3-5x faster for PaddleOCR/DocTR
3. Batch Processing: More efficient than individual files
4. Preprocessing: Improves accuracy ~5-10%, adds 200-500ms
5. Provider Selection:
   - PaddleOCR: Best speed + accuracy
   - Tesseract: Lowest memory
   - DocTR: Best for documents + tables

Troubleshooting
===============

Issue: Out of Memory
Solution: 
  - preprocess=False
  - Reduce file sizes
  - Use Tesseract instead
  - Cleanup cache

Issue: Low Accuracy
Solution:
  - Enable preprocessing
  - Try different provider
  - Use specific languages
  - Increase image resolution

Issue: Slow Processing
Solution:
  - Enable GPU
  - Use PaddleOCR
  - Use async API
  - Process in batches

Error Codes
===========

400: Invalid input (unsupported format, missing fields)
401: Unauthorized (missing token)
403: Forbidden (access denied)
404: Not found (job/batch not found)
500: Server error (processing failed)
